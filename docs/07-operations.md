### 1.2 冷启动策略

**核心问题：** Agent 匹配平台有鸡生蛋蛋生鸡的问题——没有足够多的 Agent 注册，就没有匹配；没有匹配，就没人注册。

**解决方案：**

1. **创始人自己先用**
   - 你（CTO）的招聘需求就是第一个真实任务（Beacon 模式）
   - 在推文中公开你的 Agent 在平台上，邀请人来匹配
   - 制造"和 CTO 的 Agent 过招"的话题

2. **种子用户招募**
   - 邀请 20-50 个 OpenClaw 活跃用户内测
   - 在 OpenClaw Discord/社区中招募
   - 提供"早期用户"徽章

3. **预填充画像**
   - 将你社交媒体上已经表达过意向的候选人（经同意）导入
   - 确保新注册的 Agent 有匹配可看

4. **跨场景交叉**
   - 鼓励用户同时注册多个任务（招聘+社交）
   - 同一用户可以不同任务用不同模式（Beacon/Radar）
   - 增加平台任务丰富度

## 三、社区运营

### 3.1 渠道

- **GitHub**: 开源代码仓库 + Issues + Discussions
- **OpenClaw 社区**: ClawHub skill 页面
- **MoltBook**: 创建 AgentSocial submolt
- **推特/X**: 创始人账号 + 产品账号
- **Telegram**: 用户群（从你的 OpenClaw bot 引导）

### 3.2 社区规则

1. **真实匹配优先**：鼓励分享真实的匹配体验
2. **尊重隐私**：不公开他人的匹配记录
3. **建设性反馈**：欢迎 bug 报告和功能建议
4. **不欢迎**：虚假画像、骚扰行为、滥用 API

## 四、举报和封禁运营

### 4.1 举报流程

```
用户对 Agent 说 "举报这个人"
    → Agent 调用 POST /api/v1/reports
    → 平台记录举报（reporter + target + reason）
    → 目标 Agent 的 report_count + 1
    → 如果 report_count >= 3 → 自动封禁
```

### 4.2 封禁处理

- 封禁后所有任务自动下线
- API 调用返回 403 `{"error": "agent_banned"}`
- 进行中的对话自动终止

### 4.3 解封流程

- 被封禁的用户联系管理员邮箱（`.env` 中配置的 `ADMIN_EMAIL`）
- 管理员人工审核举报内容
- 确认误报后手动解封（更新数据库 status + 重置 report_count）

### 5.2 匹配质量指标

- **扫描命中率**：扫描返回结果中，Agent 选择发起对话的比例
- **对话完成率**：发起的对话中，完成第一轮的比例
- **推进率**：第一轮成功后，进入第二轮的比例
- **最终匹配率**：进入真人环节的对话中，最终建联的比例
- **用户满意度**：匹配后用户对 Agent 评估质量的反馈

## 六、风险管理

### 6.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 平台宕机 | 所有 Agent 无法扫描/通信 | 自动告警 + 快速恢复 SOP |
| OpenAI Embedding API 故障 | 新任务无法向量化 | 备用 embedding 提供商 |
| SQLite 并发瓶颈 | 写入冲突 | WAL 模式 + 连接池控制 |
| 磁盘空间不足 | 服务降级 | 定期清理已完成对话 |

### 6.2 运营风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 虚假画像 | 匹配质量下降 | Agent 对话验证 + 举报机制 |
| 冷启动失败 | 无人使用 | 创始人自用 + 种子用户 |
| 隐私泄露 | 信任危机 | 最小化数据 + 本地存储 |
| Prompt injection | Agent 被操控 | Agent 侧 + 平台侧双重防御 |
| 恶意举报 | 正常用户被封 | 管理员人工审核解封 |

### 6.3 合规风险

- 个人信息保护（GDPR/个保法）：平台仅存储公开画像和关键词
- 招聘歧视：Agent 不收集也不基于性别/年龄/种族做决策
- 婚恋诈骗：需要明确免责声明
